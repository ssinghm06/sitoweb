Buttons

Puoi utilizzare gli stili dei pulsanti personalizzati di Bootstrap per le azioni nei moduli, nelle finestre di dialogo e altro ancora, con il supporto per molteplici dimensioni, stati e molto altro.

Esempi

Bootstrap include diversi stili di pulsanti predefiniti, ognuno con lo scopo semantico specifico, con alcune opzioni extra aggiunte per avere più controllo.

<button type="button" class="btn btn-primary">Primary</button>
<button type="button" class="btn btn-secondary">Secondary</button>
<button type="button" class="btn btn-success">Success</button>
<button type="button" class="btn btn-danger">Danger</button>
<button type="button" class="btn btn-warning">Warning</button>
<button type="button" class="btn btn-info">Info</button>
<button type="button" class="btn btn-light">Light</button>
<button type="button" class="btn btn-dark">Dark</button>
<button type="button" class="btn btn-link">Link</button>

Comunicare il significato alle tecnologie assistive.

Utilizzare il colore per aggiungere significato fornisce solo un’indicazione visiva, che non verrà trasmessa agli utenti delle tecnologie assistive, come i lettori di schermo. Dobbiamo assicurarci che le informazioni indicate dal colore siano ovvie dal contenuto stesso (ad esempio il testo visibile) oppure siano incluse attraverso mezzi alternativi, come testo aggiuntivo nascosto con la classe .sr-only.

Button tags

Le classi .btn sono progettate per essere utilizzate con l’elemento <button>. Tuttavia, è possibile utilizzare queste classi anche sugli elementi <a> o <input> (anche se alcuni browser potrebbero applicare una resa leggermente diversa).

Quando si utilizzano classi di pulsanti su elementi <a> utilizzati per attivare funzionalità in pagina (come il collasso del contenuto), invece che creare collegamenti verso nuove pagine o sezioni all’interno della pagina corrente, questi collegamenti dovrebbero essere forniti di un role=“button” per trasmettere in modo appropriato il loro scopo alle tecnologie assistive, come gli screen reader.

Link 

<a class="btn btn-primary" href="#" role="button">Link</a>
<button class="btn btn-primary" type="submit">Button</button>
<input class="btn btn-primary" type="button" value="Input">
<input class="btn btn-primary" type="submit" value="Submit">
<input class="btn btn-primary" type="reset" value="Reset">

Outline buttons

Se hai bisogno di un pulsante, ma non vuoi l’eccessivo background color che portano? Sostituisci le classi dei modificatori predefiniti con quelle .btn-outline-* per rimuovere tutte le immagini e i colori di sfondo su ogni pulsante.

<button type="button" class="btn btn-outline-primary">Primary</button>
<button type="button" class="btn btn-outline-secondary">Secondary</button>
<button type="button" class="btn btn-outline-success">Success</button>
<button type="button" class="btn btn-outline-danger">Danger</button>
<button type="button" class="btn btn-outline-warning">Warning</button>
<button type="button" class="btn btn-outline-info">Info</button>
<button type="button" class="btn btn-outline-light">Light</button>
<button type="button" class="btn btn-outline-dark">Dark</button>

Sizes

Desideri pulsanti più grandi o più piccoli? Aggiungi .btn-lg o .btn-sm per ottenere ulteriori dimensioni.

<button type="button" class="btn btn-primary btn-lg">Large button</button>
<button type="button" class="btn btn-secondary btn-lg">Large button</button>


<button type="button" class="btn btn-primary btn-sm">Small button</button>
<button type="button" class="btn btn-secondary btn-sm">Small button</button>

Puoi creare pulsanti a larghezza intera, che si estendono su tutta la larghezza del contenitore padre, aggiungendo .btn-block.

<button type="button" class="btn btn-primary btn-lg btn-block">Block level button</button>
<button type="button" class="btn btn-secondary btn-lg btn-block">Block level button</button>

Active state

I pulsanti sembreranno premuti (con uno sfondo più scuro, bordo più scuro e ombreggiatura interna) quando attivi. Non è necessario aggiungere una classe a <button> poiché usano una pseudo classe. Tuttavia, è possibile forzare lo stesso aspetto attivo con .active (e includere l’attributo aria-pressed=“true”) se si desidera replicare lo stato in modo programmatico.

Primary link Link 

<a href="#" class="btn btn-primary btn-lg active" role="button" aria-pressed="true">Primary link</a>
<a href="#" class="btn btn-secondary btn-lg active" role="button" aria-pressed="true">Link</a>

Disabled state

Per rendere i pulsanti inattivi basta aggiungere l’attributo booleano disabled a qualsiasi elemento <button>.

<button type="button" class="btn btn-lg btn-primary" disabled>Primary button</button>
<button type="button" class="btn btn-secondary btn-lg" disabled>Button</button>

I pulsanti disabilitati utilizzando l’elemento <a> si comportano in modo leggermente diverso:

  • Gli elementi <a> non supportano l’attributo disabled, quindi è necessario aggiungere la classe .disabled per renderlo visivamente disabilitato.
  • Vengono inclusi alcuni stili future-friendly per disabilitare tutti i pointer-events sui pulsanti di ancoraggio. Nei browser che supportano questa proprietà, non verrà visualizzato il cursore disabilitato.
  • I pulsanti disabilitati dovrebbero includere l’attributo aria-disabled=“true” per indicare lo stato dell’elemento alle tecnologie assistive.
Primary link Link 

<a href="#" class="btn btn-primary btn-lg disabled" role="button" aria-disabled="true">Primary link</a>
<a href="#" class="btn btn-secondary btn-lg disabled" role="button" aria-disabled="true">Link</a>

Toggle states

Aggiungi data-toggle=“button” per attivare lo stato active di un pulsante. Se si preimposta un pulsante, è necessario aggiungere manualmente la classe .active e aria-pressed=“true” al <button>.

<button type="button" class="btn btn-primary" data-toggle="button" aria-pressed="false" autocomplete="off">
    Single toggle
</button>

Checkbox and radio buttons

Gli stili .button di Bootstrap possono essere applicati ad altri elementi, come <label>, per fornire la selezione di pulsanti con stile checkbox o radio. Aggiungi data-toggle=“buttons” a un .btn-group che contiene quei pulsanti modificati per abilitare il comportamento di attivazione/disattivazione tramite JavaScript e aggiungi .btn-group-toggle per stilizzare gli <input> all’interno dei pulsanti. Si noti che è possibile creare singoli pulsanti con input o gruppi di essi.

Lo stato selezionato per questi pulsanti viene aggiornato solo attraverso l’evento di click sul pulsante. Se si utilizza un altro metodo per aggiornare l’input - ad esempio, con <input type ="reset"> o applicando manualmente le checked property dell’input - sarà necessario attivare/disattivare manualmente la classe .active sul <label>.

Si può notare che i pulsanti pre-selezionati richiedono l’aggiunta manuale della classe .active all’elemento <label> dell’input.

<div class="btn-group-toggle" data-toggle="buttons">
    <label class="btn btn-secondary active">
        <input type="checkbox" checked autocomplete="off"> Checked
    </label>
</div>


<div class="btn-group btn-group-toggle" data-toggle="buttons">
    <label class="btn btn-secondary active">
        <input type="radio" name="options" id="option1" autocomplete="off" checked> Active
    </label>
    <label class="btn btn-secondary">
        <input type="radio" name="options" id="option2" autocomplete="off"> Radio
    </label>
    <label class="btn btn-secondary">
        <input type="radio" name="options" id="option3" autocomplete="off"> Radio
    </label>
</div>
